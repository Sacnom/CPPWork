# 项目结构说明

## 📁 目录结构

```
CPPWork/
├── src/                          # 源代码目录
│   ├── Book.h                    # 图书数据模型（头文件）
│   ├── Book.cpp                  # 图书数据模型（实现）
│   ├── BookManager.h             # 管理类声明（全组共享）
│   ├── BookManager_IO.cpp        # Module A - 文件持久化
│   ├── BookManager_Add.cpp       # Module B - 录入与浏览
│   ├── BookManager_Search.cpp    # Module C - 查询
│   ├── BookManager_Edit.cpp      # Module D - 修改与删除
│   ├── BookManager_Sort.cpp      # Module E - 排序
│   └── main.cpp                  # 主程序入口
│
├── docs/                         # 文档目录
│   ├── 开发指南.md               # 整体开发指南
│   └── 开发说明/                 # 各模块专属说明
│       ├── README.md             # 文档索引
│       ├── 同学A_文件持久化.md
│       ├── 同学B_录入与浏览.md
│       ├── 同学C_查询.md
│       ├── 同学D_修改与删除.md
│       └── 同学E_排序.md
│
├── build/                        # 编译产物目录（不提交到 git）
│   └── main.exe                  # 编译生成的可执行文件
│
├── .vscode/                      # VSCode 配置
│   └── tasks.json                # 编译任务配置
│
├── CLAUDE.md                     # AI 助手项目说明
├── .gitignore                    # Git 忽略规则
└── CPPWork.code-workspace        # VSCode 工作区配置
```

## 🔧 编译和运行

在项目根目录执行：

```bash
# 编译
g++.exe -std=c++17 -g src/*.cpp -o build/main.exe

# 运行
./build/main.exe
```

或者在 VSCode 中按 `Ctrl+Shift+B` 直接编译。

## 📝 开发规范

1. **源代码** 全部放在 `src/` 目录
2. **文档** 全部放在 `docs/` 目录
3. **编译产物** 自动生成到 `build/` 目录（已在 `.gitignore` 中忽略）
4. **每人只改自己负责的模块文件**，不要动其他人的代码
5. **共享的头文件**（`BookManager.h`、`Book.h`）改动前必须全组商量

## 🚀 快速开始

1. 查看 [docs/开发说明/README.md](开发说明/README.md) 找到你负责的模块文档
2. 阅读你的专属说明文档
3. 在 `src/` 目录下编辑你负责的 `.cpp` 文件
4. 编译测试
5. 提交代码

## ⚠️ 注意事项

- `build/` 目录下的文件不要提交到 git
- 提交前先 `git pull` 避免冲突
