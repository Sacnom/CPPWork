# 开发指南（图书信息管理系统）

> 给 5 人小组的快速上手文档：看完这一份，你就知道项目长什么样、你负责哪块、代码该怎么写。

## 1. 一分钟看懂项目

这是一个命令行版的「图书信息管理系统」。整个程序就是一个 `BookManager` 类，管理一个 `std::vector<Book>`（内存里的图书列表），通过 `main.cpp` 的菜单循环调用各种功能。

数据流非常简单：

```
用户在菜单选数字  →  main.cpp 调用 manager 的某个方法  →  方法操作 books 这个 vector  →  打印结果 / 写文件
```

没有数据库、没有网络、没有图形界面。所有图书数据平时都在内存的 `books` 里，靠「保存到文件 / 从文件加载」来持久化。

## 2. 怎么编译和运行

在仓库根目录（`CPPWork/`）下打开终端，执行：

```bash
g++.exe -std=c++17 -g src/*.cpp -o build/main.exe
./build/main.exe
```

- `src/*.cpp` 会把 src 文件夹里所有 `.cpp` 一起编译成一个 `main.exe`，放在 `build/` 文件夹。**每次改完代码都重新跑这条命令**，确认能编译通过再提交。
- VSCode 里直接按 `Ctrl+Shift+B` 也能触发同样的编译任务（已在 `.vscode/tasks.json` 配好）。
- `build/` 文件夹里的产物**不要提交到 git**（`.gitignore` 已忽略）。

如果编译报错，先看是不是改坏了 `BookManager.h` 或者 `Book.h`——这两个是大家共用的，改它们会影响所有人。

## 3. 核心约定：一个类，五个文件

这是本项目最重要的设计，务必理解。

`BookManager` 这个类**只在 [src/BookManager.h](../src/BookManager.h) 里声明一次**，但它的方法实现被拆到了 5 个不同的 `.cpp` 文件里，每人负责一个。编译器最后会把它们拼回成同一个类。

| 文件 | 模块 | 你要实现的方法 | 负责人 |
|------|------|----------------|--------|
| [src/BookManager_IO.cpp](../src/BookManager_IO.cpp) | A 持久化 | `loadFromFile` `saveToFile` | 同学A |
| [src/BookManager_Add.cpp](../src/BookManager_Add.cpp) | B 录入与浏览 | `addBook` `displayAll` | 同学B |
| [src/BookManager_Search.cpp](../src/BookManager_Search.cpp) | C 查询 | `searchByTitle` `searchByAuthor` | 同学C |
| [src/BookManager_Edit.cpp](../src/BookManager_Edit.cpp) | D 修改与删除 | `deleteBook` `modifyBook` | 同学D |
| [src/BookManager_Sort.cpp](../src/BookManager_Sort.cpp) | E 排序 | `sortBooks` | 同学E |

**铁律：**

1. **只改自己那个 `.cpp` 文件**。不要把方法实现写到别人的文件里，否则会和别人冲突。
2. **不要随便改 `BookManager.h` 和 `Book.h`**。这俩是大家共享的「契约」。确实需要改方法签名时，必须先在群里和所有人商量，统一改。
3. **`main.cpp` 只管菜单调度，不要往里塞业务逻辑**。菜单和分发已经写好了，一般不用动。

目前每个模块里都是「空壳」（方法体直接 `return;`），你的任务就是把自己负责的那几个方法填上真正的逻辑。

## 4. 数据模型：Book 类

每一本书就是一个 `Book` 对象，定义在 [src/Book.h](../src/Book.h) / [src/Book.cpp](../src/Book.cpp)，全组共用，**已经写好，不用改**。它有 7 个字段：

| 字段 | 类型 | 含义 |
|------|------|------|
| `bookId` | string | 登录号（唯一标识，删除/修改时用它定位）|
| `title` | string | 书名 |
| `author` | string | 作者 |
| `categoryId` | string | 分类号 |
| `publisher` | string | 出版社 |
| `publishTime` | string | 出版时间 |
| `price` | double | 价格 |

访问方式——用 getter / setter，不能直接碰私有成员：

```cpp
Book b("B001", "C++ Primer", "Lippman", "TP312", "人民邮电", "2013", 99.0);

b.getTitle();          // 读：返回 std::string_view，可直接打印
b.setPrice(88.5);      // 写：setter 接收新值
```

⚠️ **关于 `std::string_view` 的坑**：getter 返回的是 `string_view`，它只是「借看」字符串，不拥有数据。临时打印没问题；但如果要长期保存或拼接，最好转成 `std::string`：

```cpp
std::string name(b.getTitle());   // 安全：拷一份自己的
```

## 5. 模块职责一览

每个模块负责一组相关功能，下面是简要说明。**具体怎么写代码，请查看你的专属文档**（在 [开发说明/](开发说明/) 目录）。

| 模块 | 负责人 | 核心职责 | 专属文档 |
|------|--------|----------|----------|
| **A - 持久化** | 同学A | 把内存里的图书列表存进文件 / 从文件读回内存 | [同学A_文件持久化.md](开发说明/同学A_文件持久化.md) |
| **B - 录入与浏览** | 同学B | 添加新书到内存 / 打印所有图书 | [同学B_录入与浏览.md](开发说明/同学B_录入与浏览.md) |
| **C - 查询** | 同学C | 按书名或作者关键字查找图书 | [同学C_查询.md](开发说明/同学C_查询.md) |
| **D - 修改与删除** | 同学D | 根据登录号删除或修改图书信息 | [同学D_修改与删除.md](开发说明/同学D_修改与删除.md) |
| **E - 排序** | 同学E | 按价格/书名/登录号等方式排序图书 | [同学E_排序.md](开发说明/同学E_排序.md) |

**开始开发前，务必先看你的专属文档**，里面有：
- 零基础也能看懂的概念解释
- 可以直接复制的完整代码
- 一步一步的测试方法
- 初学者常踩的坑

所有模块都围绕 `books`（`std::vector<Book>`）这个成员变量工作：
- 添加 = `books.push_back(...)` 或 `books.emplace_back(...)`
- 遍历 = `for (const auto& b : books) { ... }`
- 查找 = 遍历 + 条件判断
- 删除 = `books.erase(迭代器)`
- 修改 = 找到那本书 + 用 setter 改字段
- 排序 = `std::sort(books.begin(), books.end(), 比较函数)`

## 7. 快速开始

1. **找到你的专属文档**：打开 [开发说明/README.md](开发说明/README.md)，找到你负责的模块
2. **读完专属文档**：里面有完整代码和详细解释（10-15 分钟）
3. **编辑你的模块文件**：在 `src/` 目录下找到你负责的 `.cpp` 文件
4. **编译测试**：`g++.exe -std=c++17 -g src/*.cpp -o build/main.exe`
5. **提交代码**：`git add` → `git commit` → `git pull` → `git push`

## 8. 常见问题

- **编译报 `undefined reference`**：某个方法还是空壳没实现，或者 `src/*.cpp` 没全部参与编译。
- **改了头文件别人编译失败**：`BookManager.h` 和 `Book.h` 是共享契约，改之前先在群里说。
- **`string_view` 打印出乱码 / 崩溃**：把临时对象的 `string_view` 存起来用了，转成 `std::string` 再用。
- **load 之后数据翻倍**：`loadFromFile` 开头忘了 `books.clear()`。
- **不知道怎么写**：先看你的专属文档，里面有完整代码可以直接复制。

## 9. 协作规范

- **开工前先 `git pull`** 拉取最新代码
- **只改自己负责的 `.cpp` 文件**，不要动别人的模块
- **不要随便改共享头文件**（`BookManager.h`、`Book.h`），改之前全组商量
- **编译通过再提交**，避免把编译错误推给别人
- **提交信息写清楚**：`git commit -m "完成 Module C 按书名查询"`
- **push 前再 pull 一次**，有冲突及时沟通解决
- **不要提交 `build/` 里的文件**（`.gitignore` 已忽略）
