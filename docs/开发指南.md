# 开发指南（图书信息管理系统）

> 给 5 人小组的快速上手文档：看完这一份，你就知道项目长什么样、你负责哪块、代码该怎么写。

## 1. 一分钟看懂项目

这是一个命令行版的「图书信息管理系统」。整个程序就是一个 `BookManager` 类，管理一个 `std::vector<Book>`（内存里的图书列表），通过 `main.cpp` 的菜单循环调用各种功能。

数据流非常简单：

```
用户在菜单选数字  →  main.cpp 调用 manager 的某个方法  →  方法操作 books 这个 vector  →  打印结果 / 写文件
```

没有数据库、没有网络、没有图形界面。所有图书数据平时都在内存的 `books` 里，靠「保存到文件 / 从文件加载」来持久化。

## 2. 怎么编译和运行

在仓库根目录（`CPPWork/`）下打开终端，执行：

```bash
g++.exe -std=c++17 -g src/*.cpp -o build/main.exe
./build/main.exe
```

- `src/*.cpp` 会把 src 文件夹里所有 `.cpp` 一起编译成一个 `main.exe`，放在 `build/` 文件夹。**每次改完代码都重新跑这条命令**，确认能编译通过再提交。
- VSCode 里直接按 `Ctrl+Shift+B` 也能触发同样的编译任务（已在 `.vscode/tasks.json` 配好）。
- `build/` 文件夹里的产物**不要提交到 git**（`.gitignore` 已忽略）。

如果编译报错，先看是不是改坏了 `BookManager.h` 或者 `Book.h`——这两个是大家共用的，改它们会影响所有人。

## 3. 核心约定：一个类，五个文件

这是本项目最重要的设计，务必理解。

`BookManager` 这个类**只在 [src/BookManager.h](../src/BookManager.h) 里声明一次**，但它的方法实现被拆到了 5 个不同的 `.cpp` 文件里，每人负责一个。编译器最后会把它们拼回成同一个类。

| 文件 | 模块 | 你要实现的方法 | 负责人 |
|------|------|----------------|--------|
| [src/BookManager_IO.cpp](../src/BookManager_IO.cpp) | A 持久化 | `loadFromFile` `saveToFile` | 同学A |
| [src/BookManager_Add.cpp](../src/BookManager_Add.cpp) | B 录入与浏览 | `addBook` `displayAll` | 同学B |
| [src/BookManager_Search.cpp](../src/BookManager_Search.cpp) | C 查询 | `searchByTitle` `searchByAuthor` | 同学C |
| [src/BookManager_Edit.cpp](../src/BookManager_Edit.cpp) | D 修改与删除 | `deleteBook` `modifyBook` | 同学D |
| [src/BookManager_Sort.cpp](../src/BookManager_Sort.cpp) | E 排序 | `sortBooks` | 同学E |

**铁律：**

1. **只改自己那个 `.cpp` 文件**。不要把方法实现写到别人的文件里，否则会和别人冲突。
2. **不要随便改 `BookManager.h` 和 `Book.h`**。这俩是大家共享的「契约」。确实需要改方法签名时，必须先在群里和所有人商量，统一改。
3. **`main.cpp` 只管菜单调度，不要往里塞业务逻辑**。菜单和分发已经写好了，一般不用动。

目前每个模块里都是「空壳」（方法体直接 `return;`），你的任务就是把自己负责的那几个方法填上真正的逻辑。

## 4. 数据模型：Book 类

每一本书就是一个 `Book` 对象，定义在 [src/Book.h](../src/Book.h) / [src/Book.cpp](../src/Book.cpp)，全组共用，**已经写好，不用改**。它有 7 个字段：

| 字段 | 类型 | 含义 |
|------|------|------|
| `bookId` | string | 登录号（唯一标识，删除/修改时用它定位）|
| `title` | string | 书名 |
| `author` | string | 作者 |
| `categoryId` | string | 分类号 |
| `publisher` | string | 出版社 |
| `publishTime` | string | 出版时间 |
| `price` | double | 价格 |

访问方式——用 getter / setter，不能直接碰私有成员：

```cpp
Book b("B001", "C++ Primer", "Lippman", "TP312", "人民邮电", "2013", 99.0);

b.getTitle();          // 读：返回 std::string_view，可直接打印
b.setPrice(88.5);      // 写：setter 接收新值
```

⚠️ **关于 `std::string_view` 的坑**：getter 返回的是 `string_view`，它只是「借看」字符串，不拥有数据。临时打印没问题；但如果要长期保存或拼接，最好转成 `std::string`：

```cpp
std::string name(b.getTitle());   // 安全：拷一份自己的
```

## 5. 你的方法怎么写（按模块）

所有方法都围绕成员变量 `books`（即 `std::vector<Book>`）做文章。下面给每个模块一个起步示例，照着这个风格补全即可。

### Module A — 持久化（BookManager_IO.cpp）

把 `books` 写进文本文件 / 从文件读回来。建议每行存一本书，字段之间用一个固定分隔符（比如 `|`）隔开。

```cpp
#include "BookManager.h"
#include <fstream>
#include <iostream>

void BookManager::saveToFile(std::string_view filename) const {
    std::ofstream out{std::string(filename)};
    if (!out) { std::cout << "无法打开文件\n"; return; }
    for (const auto& b : books) {
        out << b.getBookId() << '|' << b.getTitle() << '|' << b.getAuthor() << '|'
            << b.getCategoryId() << '|' << b.getPublisher() << '|'
            << b.getPublishTime() << '|' << b.getPrice() << '\n';
    }
    std::cout << "已保存 " << books.size() << " 条记录\n";
}
```

`loadFromFile` 反过来：用 `std::getline` 逐行读，按 `|` 切分成 7 段，构造 `Book` 后 `books.push_back(...)`。注意：load 之前一般先 `books.clear()`，避免重复累加。

### Module B — 录入与浏览（BookManager_Add.cpp）

`addBook` 用 `std::cin` 依次问用户 7 个字段，造一个 `Book` 塞进 `books`：

```cpp
#include "BookManager.h"
#include <iostream>
#include <string>

void BookManager::addBook() {
    std::string id, title, author, cat, pub, time;
    double price = 0;
    std::cout << "登录号: ";  std::cin >> id;
    std::cout << "书名: ";    std::cin >> title;
    // ... 其余字段同理 ...
    std::cout << "价格: ";    std::cin >> price;
    books.emplace_back(id, title, author, cat, pub, time, price);
    std::cout << "添加成功\n";
}
```

`displayAll` 遍历 `books` 打印成一张表（注意它是 `const` 方法，只能读不能改 `books`）。`books.empty()` 时友好提示一下。

### Module C — 查询（BookManager_Search.cpp）

遍历 `books`，名字里包含关键字就打印。`const` 方法，只读。

```cpp
void BookManager::searchByTitle(std::string_view title) const {
    bool found = false;
    for (const auto& b : books) {
        if (b.getTitle().find(title) != std::string_view::npos) {
            std::cout << b.getBookId() << "  " << b.getTitle() << '\n';
            found = true;
        }
    }
    if (!found) std::cout << "没有匹配的图书\n";
}
```

### Module D — 修改与删除（BookManager_Edit.cpp）

靠 `bookId` 定位那本书。删除可以用 erase；修改就拿到那本书后用 setter 改字段。

```cpp
void BookManager::deleteBook(std::string_view bookId) {
    for (auto it = books.begin(); it != books.end(); ++it) {
        if (it->getBookId() == bookId) {
            books.erase(it);
            std::cout << "已删除\n";
            return;
        }
    }
    std::cout << "未找到登录号为 " << bookId << " 的图书\n";
}
```

`modifyBook` 同样先按 `bookId` 找到那本书，再 `std::cin` 读新值、用 setter 更新。

### Module E — 排序（BookManager_Sort.cpp）

用 `std::sort` 对 `books` 排序，传一个比较函数决定按哪个字段。可以让用户选按价格 / 书名 / 登录号排。

```cpp
#include "BookManager.h"
#include <algorithm>

void BookManager::sortBooks() {
    std::sort(books.begin(), books.end(),
              [](const Book& a, const Book& b) {
                  return a.getPrice() < b.getPrice();   // 按价格升序
              });
    std::cout << "排序完成\n";
}
```

## 6. 协作流程（避免互相覆盖）

1. **开工前先拉最新代码**：`git pull`。
2. 只动自己负责的 `.cpp` 文件，写完先本地编译（见第 2 节）确认没报错。
3. 提交时写清楚改了什么：`git commit -m "完成 Module C 按书名查询"`。
4. `git push` 前再 `git pull` 一次，有冲突当面/群里沟通解决。
5. 自己加临时调试用的文件、`main.exe` 等产物**不要提交**。

如果发现需要改 `BookManager.h` 或 `Book.h`（比如想加个新方法），**先在群里说一声，全组统一改**，否则会牵连所有人编译失败。

## 7. 常见问题

- **编译报 `undefined reference`**：八成是某个方法还是空壳没实现，或者 `*.cpp` 没全部参与编译。确认用的是 `g++ -std=c++17 *.cpp`。
- **改了头文件别人编译失败**：头文件是共享契约，改之前先沟通。
- **`string_view` 打印出乱码 / 崩溃**：多半是把临时对象的 `string_view` 存起来用了，转成 `std::string` 再用。
- **load 之后数据翻倍**：`loadFromFile` 开头忘了 `books.clear()`。
